# Capa de Servicios (Frontend) - HamBooking

La capa de servicios actúa como el puente de comunicación entre la interfaz de usuario (JavaFX) y el backend (Spring Boot). Se ha refactorizado el cliente de API para seguir principios de **Clean Code** y **Responsabilidad Única**.

## 1. ApiClient: Cliente HTTP Centralizado

El `ApiClient` es un componente crítico que encapsula toda la complejidad de las peticiones de red utilizando `java.net.http.HttpClient`.

### Mejoras Técnicas Implementadas:
- **Patrón Singleton (Bill Pugh):** Al igual que en otras utilidades, se usa una clase interna estática para garantizar una instancia única y segura en entornos multihilo, optimizando el uso de recursos de red.
- **Abstracción de Métodos (Generics):** Se han creado métodos genéricos para GET, POST, PUT y PATCH. Esto permite que cualquier controlador pueda recibir objetos DTO ya mapeados desde JSON sin preocuparse por la lógica de deserialización de Jackson.
- **Lógica de "Embudo" (Centralización):** Todas las peticiones pasan por un método privado `call`. Esto facilita futuras expansiones, como añadir un token JWT de seguridad en un solo punto para toda la aplicación.
- **Manejo de Tiempos de Espera (Timeouts):** Se han configurado tiempos de espera tanto para la conexión como para la respuesta, evitando que la interfaz se bloquee indefinidamente ante problemas de red.

---

## 2. ApiException: Gestión de Errores Tipificada

Se ha desacoplado la gestión de errores en una clase independiente, `ApiException`.

### Aspectos Destacados:
- **Transparencia de Estados:** La excepción captura el código de estado HTTP (401, 404, 500, etc.), permitiendo que el controlador tome decisiones lógicas basadas en el tipo de error (ej. redirigir al login si es 401).
- **Traducción de Mensajes:** El `ApiClient` incluye un motor de traducción que convierte los códigos de estado HTTP y los mensajes de error del backend en textos amigables y comprensibles para el usuario final en español.
- **Desacoplamiento:** Al ser una clase independiente, mejora la legibilidad de los controladores y permite una importación limpia en todo el proyecto.

---

## Resumen para la Defensa
La refactorización del `ApiClient` ha transformado un código monolítico y repetitivo en una herramienta modular y extensible. Esta arquitectura no solo previene errores comunes de red y concurrencia, sino que asegura que la lógica de comunicación con el servidor sea consistente en todas las pantallas de **HamBooking**.
